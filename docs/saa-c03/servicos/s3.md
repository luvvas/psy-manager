# S3 (Simple Storage Service)

**O que é:** armazenamento de objetos virtualmente infinito. Hospeda o frontend estático e os documentos privados.

**Quando é a resposta certa:** estáticos, backups, data lake, upload/download direto do cliente. "Servir site estático barato e escalável" = S3 + CloudFront.

## Pontos-chave
- **Block all public access** + acesso só via CloudFront (OAC) ou presigned URL.
- **Presigned URL**: delega temporariamente upload (`PUT`) ou download (`GET`) direto no S3 — o arquivo não passa pela API.
- **CORS** é necessário quando o **browser** fala direto com o S3 (`PUT`).
- **Cache-Control** por objeto controla o comportamento no CloudFront.
- Storage classes (Standard, IA, Glacier) + **lifecycle** pra reduzir custo.
- **Gateway Endpoint** (VPC) acessa S3 sem passar por NAT/internet.

## Pegadinhas
- Bucket do frontend (CloudFront baixa) **não precisa de CORS**; bucket de upload direto (browser faz `PUT`) **precisa**.
- Presigned URL só delega permissão que o emissor (IAM Role) já tem.
- "Acesso privado a estáticos" → OAC, não bucket público.
